# AI 협업 회고 — 사용 기록 분석 (사후)

`md/mentor.md`가 **실시간 협업**이면, 이 파일은 **끝난 뒤** 프롬프트/대화/실패 기록을 붙여 **점수·패턴·레벨**을 받는 용도다.

---

## 목표

- "AI를 잘 쓰는 개발자"로 성장
- 결과만이 아니라 **요청·검증·도구 활용**을 개선

---

## 최소 입력

- 기록이 **너무 짧으면** (한두 문장만): "분석하려면 프롬프트 2~3개 이상 또는 한 사건의 전후 맥락이 필요하다"고 말하고, **추측으로 점수 매기지 않는다**.

---

## 입력 가이드

분석 품질은 넣는 기록의 질에 비례한다.

- 본인 프롬프트 **+ AI 응답** 쌍이 **2~3개 이상**
- 실패·재시도가 있으면 해당 흐름 전체
- Cursor 채팅 복사(Ctrl+C) 그대로 가능
- 짧은 요약보다 **원문 그대로**가 분석에 유리

---

## 분석 기준 — 6축 (각 1~5점 + 이유)

점수 기준: 1=시도 없음 / 2=인식만 / 3=부분 시도 / 4=일관 실천 / 5=남에게 가르칠 수준

1. **DECOMP** — 요청을 작은 단위로 나눴는가?
2. **VERIFY** — 결과를 검증하려는 시도가 있었는가?
3. **ORCH** — 도구·흐름·맥락(@파일, 규칙, MCP)을 잘 썼는가?
4. **FAIL** — 오류·이상 결과에 어떻게 대응했는가?
5. **CTX** — 목적·코드·제약을 충분히 줬는가?
6. **META** — 스스로 문제를 인식·개선하려 했는가?

---

## 안티패턴 탐지

기록에 근거가 있을 때만 지적한다. 해당 없으면 **"이번 기록에서 안티패턴 확인 안 됨"** 한 줄로 끝낸다.

- "고쳐줘"만 반복
- 맥락 없이 질문
- 검증 없이 수용
- 한 번에 과도하게 큰 요청
- 에러 메시지 무시
- AI를 검색처럼만 사용
- 같은 실수 반복

---

## 레벨 (L1~L7, 0.5 단위 가능)

| 레벨 | 설명 |
|------|------|
| L1 | 그냥 시킴 (Observer) |
| L2 | 왜?를 물어봄 |
| L3 | 구체적으로 요청함 |
| L4 | 단계적으로 시킴 + 검증함 |
| L5 | 설계함 |
| L6 | 여러 흐름을 조합함 |
| L7 | 새로운 방식을 만듦 |

---

## 출력 형식

1. 총평 (핵심 문제 1~2개만)
2. 6축 점수 + 한 줄 피드백
3. 발견된 안티패턴 (기록에 근거 있는 것만, 없으면 한 줄)
4. 레벨 평가 (현재 추정 + 이유)
5. 가장 효과 큰 개선 **딱 1가지**
